AD-HOC PROMPTS
==============

  ralph prompt <file> [max_iterations]

Run a custom prompt through ralph's iteration loop — no planner, no task
selection, no implementation plan. Your prompt file replaces the built-in
plan/build prompts with a custom workflow while still leveraging the loop's
iteration, logging, and exit-signal machinery.

WHEN TO USE
-----------

  - Work that benefits from multiple iterations but doesn't fit the
    planner's task-decomposition model (spec review, test analysis,
    document assembly, codebase audits)
  - You want to define your own iteration strategy, progress tracking,
    and completion criteria
  - Quick one-shot jobs that just need one iteration

WHEN NOT TO USE
---------------

  - The work has phases or ordering constraints → ralph plan --process
  - The work is defined by target-state specs → ralph plan
  - You need the planner to decompose the work into tasks → use a planning mode

WRITING A MULTI-ITERATION PROMPT
---------------------------------

  Each iteration starts with fresh context — the agent has no memory of
  prior iterations. Effective multi-iteration prompts include:

  - A progress tracking file the agent reads first and updates last,
    serving as durable memory across iterations
  - A "fresh context" reminder so the agent treats repo files as the
    sole source of truth
  - Iteration sizing guidance (how much work per iteration)
  - Clear rules about what the agent should and should not modify

  CRITICAL: Your prompt MUST instruct the agent to output exactly
  <promise>COMPLETE</promise> when all work is done. This is the only
  way the loop knows to stop. Without it, ralph will keep iterating
  until max_iterations is reached.

  The built-in prompts in .ralph/prompts/ are good starting templates.
  Study .ralph/prompts/build.md or .ralph/prompts/plan.md for structure
  and conventions.

EXAMPLES
--------

  Single iteration:

    ralph prompt .ralph/prompts/my-refactor.md

  Multi-iteration with a cap:

    ralph prompt .ralph/prompts/adhoc-spec-review.md 20

SHIPPED PROMPTS
---------------

  Ralph ships two ad-hoc prompts ready to use:

  Test analysis — study your test suite to enhance specs with business
  rules, edge cases, and validation logic the specs are missing:

    ralph prompt .ralph/prompts/test-analysis.md 10

  Process spec review — study the codebase to find gaps, ordering
  problems, and missing dependencies in your process specs before
  running ralph plan --process:

    ralph prompt .ralph/prompts/process-spec-review.md 10

BUILDING YOUR OWN PROMPT LIBRARY
---------------------------------

  Since .ralph/ is committed to your project's git repo, custom prompts
  you add to .ralph/prompts/ are versioned and shared with your team.
  Over time, you build up a library of reusable prompts tailored to your
  project — test sweeps, code audits, spec drafters, doc generators.

  Convention: prefix ad-hoc prompts with adhoc- to distinguish them from
  ralph's system prompts (plan.md, build.md, etc.):

    .ralph/prompts/adhoc-test-sweep.md
    .ralph/prompts/adhoc-discovery.md
    .ralph/prompts/adhoc-doc-generator.md

  This makes them easy to list (ls .ralph/prompts/adhoc-*) and signals
  they are meant for ralph prompt, not used internally by ralph.

CUSTOM PROMPT IDEAS
-------------------

  Codebase discovery — survey a project before writing specs:

    ralph prompt .ralph/prompts/adhoc-discovery.md

  Test sweep — agent-driven QA pass after a build cycle:

    ralph prompt .ralph/prompts/adhoc-test-sweep.md 10

  Spec drafting — have the agent draft specs from a description:

    ralph prompt .ralph/prompts/adhoc-draft-specs.md

  These are abbreviated sketches. Real prompts need significantly more
  structure — fresh-context reminders, progress tracking, scope constraints,
  verification steps, and commit discipline. Use the shipped prompts in
  .ralph/prompts/ as templates for the right level of detail.
